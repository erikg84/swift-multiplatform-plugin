# 03 — iOS Manifest Generation (Priority #2)

## Problem
swift-java doesn't support iOS as a platform. When `xcodebuild` tries to build the Swift package for iOS, it fails because it can't resolve swift-java for the iOS target.

The prototype works around this with a shell script that swaps Package.swift before building. This is fragile and external to the build system.

## Solution: GenerateIosManifestTask + PackageSwiftGenerator

The plugin generates a temporary iOS-only Package.swift at build time by parsing the original and stripping Android-only dependencies.

### PackageSwiftGenerator.kt


### Approach: Template-Based (Simpler, More Reliable)

Rather than parsing Swift source (which is complex and error-prone), generate from a template using the extension config:

```kotlin
fun generateIosManifest(
    outputFile: File,
    moduleName: String,
    iosPlatformVersion: String,
    dependencies: List<Pair<String, String>>,  // [(url, version)]
) {
    val depsBlock = dependencies.joinToString("\n        ") { (url, ver) ->
        ".package(url: \"$url\", from: \"$ver\"),"
    }
    val prodDepsBlock = dependencies.joinToString("\n                ") { (url, _) ->
        val name = url.substringAfterLast("/").removeSuffix(".git")
        ".product(name: \"${name.capitalize()}\", package: \"$name\"),"
    }

    outputFile.writeText("""
        |// swift-tools-version: 6.2
        |// Generated by swift-multiplatform Gradle plugin — iOS build only
        |import PackageDescription
        |let package = Package(
        |    name: "$moduleName",
        |    platforms: [.iOS(.v${iosPlatformVersion.substringBefore(".")}), .macOS(.v12)],
        |    products: [
        |        .library(name: "$moduleName", targets: ["$moduleName"]),
        |    ],
        |    dependencies: [
        |        $depsBlock
        |    ],
        |    targets: [
        |        .target(name: "$moduleName", dependencies: [
        |            $prodDepsBlock
        |        ]),
        |    ]
        |)
    """.trimMargin())
}
```

### Extension Config for iOS Dependencies

The extension needs a way to declare which dependencies are iOS-safe:

```kotlin
swiftMultiplatform {
    // These dependencies are included in both iOS and Android builds
    commonDependencies {
        spm("https://github.com/Swinject/Swinject.git", "2.10.0")
    }

    // These are Android-only (stripped for iOS builds)
    android {
        dependencies {
            spm("https://github.com/swiftlang/swift-java.git", "0.1.2")
        }
    }
}
```

OR simpler — auto-detect by stripping known Android-only deps:

```kotlin
ios {
    stripDependencies("swift-java")  // default: ["swift-java"]
    stripPlugins("JExtractSwiftPlugin")  // default: ["JExtractSwiftPlugin"]
}
```

### GenerateIosManifestTask

```kotlin
abstract class GenerateIosManifestTask : DefaultTask() {
    @get:InputFile
    abstract val originalManifest: RegularFileProperty

    @get:OutputFile
    abstract val generatedManifest: RegularFileProperty

    @get:Input abstract val moduleName: Property<String>
    @get:Input abstract val iosPlatformVersion: Property<String>
    @get:Input abstract val stripDependencies: ListProperty<String>

    @TaskAction
    fun generate() {
        // Backup original
        val backup = File(originalManifest.get().asFile.parentFile, "Package.swift.build-backup")
        originalManifest.get().asFile.copyTo(backup, overwrite = true)

        // Generate iOS-only manifest
        PackageSwiftGenerator.generateIosManifest(...)

        // Write to the ORIGINAL location (xcodebuild reads from project root)
        originalManifest.get().asFile.writeText(generatedManifest.get().asFile.readText())
    }
}
```

### RestoreManifestTask

Runs after iOS build completes (success or failure):

```kotlin
abstract class RestoreManifestTask : DefaultTask() {
    @TaskAction
    fun restore() {
        val backup = File(project.projectDir, "Package.swift.build-backup")
        if (backup.exists()) {
            backup.copyTo(File(project.projectDir, "Package.swift"), overwrite = true)
            backup.delete()
        }
    }
}
```

### Task Graph

```
generateIosManifest  (swaps Package.swift to iOS-only)
    ↓
buildIosDevice + buildIosSimulator
    ↓
restoreManifest  (restores original Package.swift)
    ↓
assembleXCFramework
```

The restore task is also wired as a `finalizedBy` on the build tasks so it runs even on failure.
